# Network Troubleshooting – TCP/IP & DNS Issue Resolution

## Objective
To diagnose and resolve a network connectivity issue using structured troubleshooting techniques.

---

## Scenario
A user reported inability to access websites despite being connected to a network. The objective was to identify the root cause and restore connectivity.

---

## Troubleshooting Process
- Checked physical and network connection status  
- Verified IP configuration settings  
- Tested connectivity using ping to local and external addresses  
- Analysed DNS resolution behaviour  
- Isolated the issue to DNS configuration failure  

---

## Findings
- System had incorrect DNS settings  
- Unable to resolve domain names into IP addresses  
- Network connection was active but not functional for web access  

---

## Resolution
- Updated DNS settings to a valid server  
- Flushed DNS cache  
- Retested connectivity successfully  

---

## Outcome
- Internet access restored  
- System able to resolve and access external websites  

---
